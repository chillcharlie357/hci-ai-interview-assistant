# MVP 目标

## 产品目标

实现一个可演示的面试智能辅助 MVP，覆盖以下闭环：

1. 输入候选人简历摘要、岗位 JD、面试目标。
2. 系统解析输入并生成结构化面试问题。
3. 数字人面试官按问题逐个提问。
4. 系统记录候选人回答、表达指标和关键事件。
5. 系统在候选人授权后记录浏览器端摄像头观察信号和内存关键帧。
6. 面试结束后生成完整智能纪要。

## MVP 范围

第一版采用 Python 实现后端和核心面试逻辑，TypeScript 实现前端 UI。MVP 支持 OpenAI-compatible LLM 问题/纪要增强，也支持浏览器端摄像头观察信号。MVP 不接入真实 WebRTC 会议、屏幕共享、OCR、高精度视觉推理和实时数字人渲染。数字人提问和面试会话均用可替换接口封装，后续可接入 LiveKit、LiveKit Agents、Pipecat 或 LiveTalking。

## 核心成功标准

- 可以从简历/JD生成不少于 6 个面试问题。
- 问题覆盖技术能力、项目经验、实现能力、行为表现和追问。
- 数字人面试官可以用文本和前端 TTS 逐题提问。
- 可以记录每道题的回答内容、用时、字数和填充词数量。
- 可以开启摄像头并记录非语言观察信号，关键帧默认只保存在内存 session。
- 未配置 OpenAI 环境变量时可规则 fallback，并返回 `llm_status: fallback`。
- 可以在面试后生成 Markdown 风格纪要。
- 招聘端可以查看生成的完整测试题目，包含追问和观察点。
- 可以按报告可见性下载 Markdown 面试结果。
- Python 核心逻辑和 TypeScript 前端关键流程有自动化测试覆盖。

## 非目标

- 不在 MVP 中实现真实视频会议和屏幕共享。
- 不在 MVP 中实现真实数字人唇形同步。
- 不在 MVP 中实现高精度摄像头非语言识别、情绪识别或人格推断。
- 不在 MVP 中实现真实视频抽帧、OCR 和 WhisperX。
- 不输出录用或不录用结论。
